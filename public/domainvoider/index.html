DomainVoider


DomainVoider started as a project to provide two filters, mainVOID & modVOID, to AdGuard Pro users. Each was built from a hodgepodge of sources, like so many other filter projects. Each filter provided users with privacy & security benifits that protected entire iDevices. It has continued to evolve since 2016 & now DomainVoider combines a modified version of AdGuard's DNS filter with a list of phishing domains licensed from Hardenize.com & malware URLS from Urlhaus provided data.

Privacy & security have become DomainVoider's focal point, & to better create each build, one filter, made for DNS-based protection, is produced.

Moreover, DomainVoider is also for use with uBlock Origin or AdGuard for Firefox. When used with either extension, the included CSP rules frame-ancestors 'none', upgrade-insecure-requests & block-all-mixed-content help to increase web security without breaking websites. A Blocked by Content Security Policy notice used to be displayed on websites that violated any of the four CSP rules. Now, Firefox, e.g., displays various warnings depending on the rule broken. Our four main Content Security Policy rules:

frame-ancestors 'none'

helps to protect against clickjacking,

upgrade-insecure-requests

instructs user-agents to upgrade connections to capable hosts from one that's unencrypted http:// connection to an encrypted connection using (hopefully) at least TLS1.2 as TLS1.3 is likley nearing its EOL if it follows a similar timleline as previous iterations.

block-all-mixed-content

instructs user-agents to block the use of resources loaded over http:// when requested by sites that do use https://,

object-src 'none'

helps to stop plugins such as Java & Flash that attempt to load & execute using object, embed or applet tags.

The CSP rule object-src does not interfere with browser extensions such as AdGuard, uBlock Origin, Decentraleyes, ClearURLs & HTTPS Everywhere. The CSP rule object-src restricts plugins, none of which are necessary for any modern website to function.
