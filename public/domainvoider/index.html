DomainVoider

DomainVoider is dedicated to providing AdGuard & AdGuard Pro for iOS users with a custom, privacy & security focused, simplified ad filter. DomainVoider combines a modified version of AdGuard's DNS filter with a list of phishing domains licensed from Hardenize.com

DomainVoider is also for use with uBlock Origin or AdGuard for Firefox. When used with either extension, the included $CSP rules frame-ancestors 'none', upgrade-insecure-requests & block-all-mixed-content help to increase web security without breaking websites. A Blocked by Content Security Policy notice used to be displayed on websites that violated any of the four CSP rules. Now, Firefox, e.g., displays various warnings depending on the rule broken. Our four main Content Security Policy rules:

  1. frame-ancestors 'none' helps to protect against clickjacking,
  2. upgrade-insecure-requests instructs user-agents to upgrade connections to capable hosts from one that's unencrypted http:// connection to an encrypted connection using (hopefully) at least TLS1.2 as TLS1.3 is likley nearing its EOL if it follows a similar timleline as previous iterations.
  3. block-all-mixed-content instructs user-agents to block the use of resources loaded over http:// when requested by sites that do use https://,
  4. object-src 'none' helps to stop plugins such as Java & Flash - a source of exploits since its initial release by Adobe - that attempt to load using object, embed or applet tags.
  5. An experimental $CSP rule require-trusted-types-for is included in the current release.

The $CSP rule object-src does not interfere with browser extensions such as AdGuard, uBlock Origin, Decentraleyes, ClearURLs & HTTPS Everywhere. The $CSP rule object-src only restricts plugins, none of which are necessary for any modern website to function.
